## Reference

### Summary
Corbin Crutchley (Staff Software Engineer at Descript) shares his experience with hybrid/remote teams and the tradeoffs between in-person and remote work for engineering teams. Key themes: documentation importance, flow state, culture building, and accountability challenges.

### Key Takeaways

**1. Political Adaptability (In-Office Advantage)**
- In-office work can be disproportionately advantageous for understanding internal politics
- Remote workers are relegated to the "buttoned-up" view in meetings
- Casual conversations and idea sessions happen in water-cooler moments
- Junior engineers with in-office visibility can accelerate careers faster

**2. Documentation Needs**
- Office-first communication loses information easily
- "Bus factor" drops significantly - knowledge sits in one person's memory
- Remote work requires radically transparent communication
- Remote meetings are easier to record, transcribe, and index for search
- Documentation is costly for in-office work, so many orgs skip it

**3. Flow State**
- Deep work requires uninterrupted focus time
- In-office: colleagues can break your flow with casual chat
- Remote: you control what notifications break through
- Critical for engineers in novel problem spaces and debugging

**4. Culture and Remote Work**
- Remote requires deliberate, inclusive team-building
- Organic "drop by desk" bonds don't happen naturally
- Solutions: open-office policies (virtual), lunch-and-learns, async social chats, jokes/memes
- Activities should be optional - respect work/life separation preferences

**5. Accountability Challenges**
- Remote work can enable "bed to desk to nap" patterns
- Harder to have peer accountability
- Monitoring software instills distrust and can be gamed
- Solution: stay on top of team communication, check ticket progress/estimations
