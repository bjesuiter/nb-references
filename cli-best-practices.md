## My Reference

### Summary

CLI Guidelines (clig.dev) provides comprehensive best practices for designing and building command-line interfaces. The gold standard for CLI UX and developer experience.

**Key topics covered:**
- Command structure and naming conventions
- Output formatting and colors
- Error handling and messages
- Configuration and flags
- Help text and documentation
- Cross-platform considerations
- Performance and progress indicators

### Content

Useful for:
- Designing new CLI tools
- Improving existing command-line applications
- Following industry-standard CLI patterns
- Creating consistent CLI experiences
- Documentation and help text design

**Why it matters:** Good CLI tools feel intuitive, provide helpful errors, and follow consistent patterns that users expect.
